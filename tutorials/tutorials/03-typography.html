Typography on the Web

Best Practices

Things to Remember

  • Use fonts that show enough contrast
  • Use fonts that cater to your design
  • Use fonts that are legible at a small size
  • Use one or two fonts at a time for consistency

Google Fonts

Google Fonts is a free resource with hundreds of downloadable fonts with easy access to embedding for your site. I would say fonts from Google are most used in my time at an agency because of how easily accessible it is.

https://fontpair.co/ is a great resource to help you pair Google fonts. Think back to fundamentals of typography.

How to Use Google Fonts

  1. Browse the font list or search for a font

  2. Open the font by clicking on the font cell

  3. On the top right of the page there's a blue link to download the family

  4. On the right side of each font style is an option to "Select this style."

  5. Once you select a style, the sidebar will pop up. You can select multiple styles before choosing to embed (HTML or CSS)

Adobe Typekit

Adobe Typekit is a font service free for those who have a Creative Cloud membership. Use this if you plan on keeping your membership after your graduate (you should).

How to Use Adobe Typekit

  1. Browse the font list or search for a font

  2. Open the font by clicking on <*/>

  3. Create a project and title it

  4. Click Createe

  5. Once the HTML import pops up, copy it. There is an option to @import to CSS

Embedding fonts

There are two ways to embed your fonts into your website. You can link to your HTML or import to your CSS.

  1. Link in HTML

    <*link href="https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;600;700;800&display=swap" rel="stylesheet">

    This would go in the head of your document, typically under your CSS link.

  2. Import into CSS

    @import url('https://fonts.googleapis.com/css2?family=Open+Sans:wght@300;400;600;700;800&display=swap');

Styling type

You should start with the general styles:

<*body>, <*h1> - <*h6>, <*a>

Build on top of the general styles for your buttons, navigation links, and paragraphs.

Your <*body> font will most likely match your <*p>.

Your <*a> should have button styles applied to them.

Note: <*button> will be used in forms only.

All links should have hover effects.

body {
    font-family: "Open Sans", sans-serif;
    font-size: 16px;
    line-height: 24px;
    color: #000000;
}

h1, h2, h3, h4, h5, h6 {
    font-family: "Open Sans";
    font-weight: 600;
}
                        
a {
    font-weight: 600;
    color: #ff0000;
    text-decoration: none;
}
                        
a:hover {
    opacity: .6;
}

  1. I defined the font styles of the body. I chose what would be the smallest type, which would be our body copy.

  2. Keep this original style general. The font weight should have more contrast than your body copy, which you've already designed.

  3. I defined the general links of the body. Choose a style that has a little more weight and an alternate color.

  4. I defined the hover state for links. I chose opacity here because if I were to just change the font color, I would have to adjust so much more on the button classes.

h1 {
    font-size: 3rem;
    line-height: 2.5rem;
}

h2 {
    font-size: 2.5rem;
    line-height: 2rem;
}

.btn {
    display: inline-block;
    padding: 2rem 1rem;
    border: 1px solid transparent;
}

.filled {
    background: gray;
    color: white;
}

.outline {
    border: 1px solid gray;
    color: gray;
    background: white;
}

  1. Your h1 should be the largest heading you use.

  2. I used rems for units here because they are relative to the body font. In this case, 1rem = 16px. If we ever decide to bump up the body text, eveything else with a rem unit will increase in size (or decrease, if that's the choice).

  3. The .btn class will be added to the class of your buttons first, then you add a space and add which style you'd like. You don't have to have both--it just depends on your design. But this is a good start. <*a class="btn outline">

  4. Setting the display to inline-block allows the link to stay within the bounds of its parent and will accept paddings and margins.

.Button vs Button

If you want to have a button that will go to another page on your website, use an anchor tag with a button class such as <*a class="btn">. You can separate your different styles of buttons by creating other classes and adding them after your base class.<*a class="btn outline">

If you want users to sign up for a newsletter or contact you, use the <*button> tag within a form.

These buttons can be styled similarly, but remember they have very different functions.

Icons

There are several resources to get icons. These are just a few recommendations.

  • Envato Elements is a paid membership with tons of resources, especially UI icons. They have a student discount.

Lists

There are several places you can use an unordered and an ordered list:

  • Navigations
  • Footers
  • Restaurant Menus
  • Recipes
  • Tutorials
  • etc.

While these might not look the same, they are definitely lists.

Lists in HTML

<*ul> is an unordered list. Each bullet point is referred to as a marker in CSS.

<*ol> is an ordered list. The markers appear as numbers or letters and you can choose which one.

<*li> is a list item placed inside of an unordered and ordered list.

You must have list items inside your unordered/ordered list.

<*ul>
    <*li>Item 1<*/li>
    <*li>Item 2<*/li>
    <*li>Item 3<*/li>
<*ul>

<*ol>
    <*li>Item 4<*/li>
    <*li>Item 5<*/li>
    <*li>Item 6<*/li>
<*ol>

Lists in CSS

Lists have inherited or default CSS properties.

ul {
    display: block;
    list-style-type: disc;
    margin-top: 1em;
    margin-bottom: 1em;
    margin-left: 0;
    margin-right: 0;
    padding-left: 40px;
}

You can reset these properties in your stylesheet for navigations and footers.

ul {
    list-style-type: none;
    padding-left: 0;
    margin: 0;
}

We'd have to use specificity if we'd like to reset unordered lists in navigations, so you'd write the selector nav ul. This way your other lists won't be affected and have their inherited CSS.

How to style general lists

  • list-style-type specifies the type of list item marker. Click here for several options.

  • list-style-image specifies an image as the list item marker. You'd write the url.

  • list-style-position specifies the position of the list-item markers. There are two options.

    list-style-type: square;

How to Style Navigations

  1. Reset your <*ul>

  2. Reset your <*li> to display inline-block

  3. Don't forget to add your anchor tags!

You will usually have your navigation at the very top of your page, which is your <*header>.

Inside of your <*header> will be your <*nav>.

Inside of your nav will be your <*ul>.

Why so much stacking? You might be inclined to put a hero image where your header is. You might not, but it's good practice to have the proper semantics in order! Proper semantics are when you use the right tags named after the sections on your page. <*header>, <*nav>, <*section>, <*p>, etc. (as opposed to using only divs or spans).

ul {
    list-style-type: none;
    padding-left: 0;
    margin: 0;
}

li {
    display: inline-block;
}

Now you will add more CSS to make your nav items fit correctly. You might have to create different classes.

How to Style Footers

  1. Reset your <*ul>

  2. Set the first <*li> to be a heading.

  3. Each column on your footer should be a <*ul>

  4. Any form elements should be in a <*form>. Note that for now, you dont put anything in the form.

  5. Don't forget to add your anchor tags!

ul {
    list-style-type: none;
    padding-left: 0;
    margin: 0;
}

li:first-of-type {
    font-weight: bold; /* or whatever number */
}

Project 1B: The Header & Footer

Now that you have a basic structure for your site in place, it's time to start working on the overall style and design. We'll begin with making sure our header and footer is coded.

  • For this stage, you are only required to worry about coding the header and footer on your homepage

  • Set up your Google fonts if you haven't already.

    • Play Fair Display Regular for headings

    • Open Sans Light, Semibold, Bold for body copy

  • Use the UL technique discussed Tutorial 3

    • Remember specificity: you don't want to style every UL on your site or every link act like your nav. How do you make sure it only affects that section?

  • Remove any forced heights only in your header and footer. Remember, we try not to use heights unless absolutely necessary. Let your content and padding determine the height of your divs and other elements.

  • Make sure your logo links back to the home page

  • Don't forget hover effects

  • Use an input field for the email sign up

  • Make sure "Submit" in the footer is a button. I will teach you how to make it an arrow in a later tutorial, so sit tight on that.
