Images

Images can be used in many different scenarios on your website. In this tutorial, we will touch on a few instances that will be helpful to you.

The img tag displays an image on your site.

<*img src="url" alt="Description">

We know that the src attribute is where the url or path to the image is. The alt attribute is a short description describing the image in case the image doesn’t show up or the page is being read by screen readers. Keep your descriptions short, but clear so the user understands what is going on in the image.

Displaying images in HTML

You can start with the <*figure> tag. Inside of it will be your <*img> and, if necessary, a <*figcaption>

<*figure>
    <*img src="" alt="Marinara Sauce">
    <*figcaption>This is a caption.<*/figcaption>
<*/figure>

The only places you might not use this are instances of your logo and instances of icons. PLEASE keep that in mind.

figure {
    display: block;
    margin-top: 1em;
    margin-bottom: 1em;
    margin-left: 40px;
    margin-right: 40px;
  }

The <*figcaption>’s default CSS is only that it’s a block element. It defines the image, but it’s not necessary.

You can also display the <*img> by itself, but remember, using proper semantics now might help you in the future when you know more. What if you want a really cool hover effect on an image where a caption is revealed? Your knowledge of <*figure> and <*figcaption> will help!

Background Images

If you want to display an image as a background, you’ll have to use CSS.

All you have to do is decide where it belongs. Is it in your section (full width of browser)? Is it in your article (within the confines of your container)?

Either way, a background image will be defined in your CSS.

Let’s say, I have a section as follows.

<*section id="location">
    <*article class="container clearfix">
        <*div class="float-left">
    <*/article>
 <*/section>

The <*section> is a full width element that holds the <*article>, which in this case, is our container. In your css, instead of adding a background image to all sections, you can specifically set it to #location

#location {
    background-image: url("/images/bg.jpg");
    background-size: cover;
    background position: center;
    background-attachment: fixed;
    background-repeat: no-repeat;
    }
  1. I set the url to whatever path is necessary to find the image.
  2. The background size is set to cover which means the background image will always fill up the space of the element it's on. Check out this resource
  3. The background-position has several values available. I would check out this resource to help you determine what you can use.
  4. The background-attachment scrolls with he page on default, but you can place it fixed to the page so the elements scroll independently from it. It gives a really cool parallax effect without using javascript. I would check out this resource.
  5. The background-repeat allows you to set the repetition of the image. If it’s a hero image, it will most likely not be repeated. But background patterns will be repeated. Check out this resource.

There are other properties for setting the background and you can check them out here. We will learn more about other background properties in the coming tutorials. There’s also a shorthand property which is useful to learn to cut down your time writing CSS.

Optimizing Images

In your Project 1 image files, there's a .tiff. We know that's not web safe. You can use photoshop to create the optimized file.

  1. Image > Mode: Make sure the image is RGB color mode

  2. Image > Image size: Make sure the image is an appropriate size (nothing outrageous like 2000-5000px) and at 72DPI

  3. File > Export > Save for Web > PNG-24: Save to your images folder
