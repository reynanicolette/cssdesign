FTP

What is FTP?

As web designers you will need to know how to get your work uploaded to a web server. The process to transfer files to a web server is often referred to as FTP – which is an acronym that stands for "file transfer protocol". Essentially all it means is that you are uploading files to a web server. 

FTP Programs

In our graphics labs computers should have a program named Fetch installed on them. It is one of the easiest FTP programs to use on a macintosh computer. Students may apply for a free license for educational use.

Although we typically teach FTP by using Fetch, please know that there are many free programs that enable you to FTP from your own computer. It is wise this week to find a FTP program that will enable you to transfer your files this semester. Here are a few popular programs:

  • Cyberduck
  • Smartftp
  • Fireftp
  • Filezilla

Setting up your files for FTP

Your folder set up should be as follows

  • Main Folder

    • CSS Folder

      • style.css

    • Images Folder

      • image1.jpg

      • image2.jpg

      • image3.jpg

    • index.html

As long as you can see the changes you’ve made to your CSS, you should be able to upload to your preferred FTP server and see your page without issues.

Troubleshooting FTP

You all are likely new to FTP'ing and unless you have done FTP before it may seem daunting. We have a few suggestions for you to help you learn to troubleshoot if you run into troubles this semester. 

Watch for typos

If you are having login troubles always first check to make sure you haven't made a typo. It's easy to do so and is a common error. Additionally be sure you are typing in all lowercase, with no spaces, and no special characters. Any typos could lead to errors when you attempt to login. 

Always have an index.html file

The web server will search for a file named index.html so always ensure your html begins with an index page. It will be the first page that loads.

Check for file size

Graphics files can get big, especially if you ignore optimization of images. If you exceed your quota (50mb) you will get an error. If you go over your file quota be sure you haven't ftp'd any large files or unnecessary files. You will want to remove files that are too large or taking up space if you get into quota trouble. Please only remove work that has been graded to make room for new files. 

FTP limitations

Our server is a little picky with how many of you can ftp at the same time. This is typically only a problem on demo day, however, if you find several students are trying to FTP at the same time we have found sometimes the host will temporarily block our ip address if too many people login at the same time. If this happens just log out and give it 5-10 minutes and typically you will be able to login again.

Do not try to login incorrectly more than 10 times

We know you all will try to login many times and if you have troubles try your best to work through them but please stop trying as you get close to 10 incorrect attempts. By this point the server may temporarily block your ip address if you try to login incorrectly too many times. Instead contact your instructor to request support with your ftp.
