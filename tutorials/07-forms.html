Forms

Forms collect information you want to gather from the user. They fill out the form, submit it, and then it is processed any way youâ€™d like. While you may not need to know how to make a form function, it is good to know and understand how forms are designed.

Interacting with forms

  1. <*form> introduces a form. This is a block element.

  2. <*input> displays an input for a user to input information & submit it. This is an inline element.

  3. <*label> allows users to understand what the input is for. This is an inline element.

Styling Inputs

The form elements are too small for thumbs on mobile devices. Plus the elements are so squished together that it can be hard to read on screens, so styling them and adding more space creates a better design. 

#form-ex input {
    outline: 0;
    border: 0;
    border-radius: .25rem;
    background: #e3e3e3;
    width: 100%;
    padding: 1rem;
    margin: 0 0 2rem 0;
    font: 400 1rem/1rem "Open Sans";
}

#form-ex input:hover {
    opacity: .6;
}

#form-ex label {
    display: block;
    margin-bottom: .5rem;
    font-weight: 500;
}

#form-ex input[type="submit"] {
    font-weight: 600;
}

  • The first thing you will want to do is reset your inputs. I set the outline to zero. The outline is the blue outline that shows up when you focus or click inside of an input.

  • I set the border to 0. Sometimes designs call for borders, but in this case, I didn't want one.

  • I set the border-radius to .25rem. The border-radius affects the corner roundness.

  • I set the background-color to a gray color.

  • I set the width to 100%. This ensures the elements fill the grid as much as possible.

  • I set the padding to 1rem. This creates an open design. The edges of the input aren't too close to the placeholder.

  • I set the margin on the bottom to 2rem and 0 everywhere else to make sure the space is only distributed at the bottom. I chose a rather large value so that the inputs wouldn't be so close to the labels below it. Remember spacing is good.

  • I set the fonts to match my body font. The default font is Arial, so it's important to declare for the inputs.

  • I set the hover on the input. I wanted the interaction to be clear. The hover state tells the user: You can click me!

  • I set the label o be a block element so it is on a separate line from the input below it.

  • I set the label's and submit's font weight to 500, 600 (medium, semibold) so it stands out and establishes hierarchy from the input below it.
